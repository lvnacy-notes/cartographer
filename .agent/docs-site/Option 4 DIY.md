# Option 4: DIY Documentation Site Setup Guide

This guide shows you how to quickly create a simple documentation site that includes your Storybook alongside custom pages.

## Project Structure

```
your-project/
├── docs-src/               # Your custom documentation source
│   ├── index.html         # Home page
│   ├── getting-started.html
│   ├── api.html
│   ├── styles.css         # Shared styles
│   └── assets/
│       ├── logo.svg
│       └── favicon.ico
├── storybook-static/      # Generated by Storybook (gitignored)
├── docs/                  # Final build output (gitignored)
└── .github/
    └── workflows/
        └── storybook.yml
```

## Step 1: Create Your Documentation Pages

Create a `docs-src` folder with a basic home page:

**docs-src/index.html:**
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Your Plugin Name - Documentation</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <nav class="sidebar">
        <h1>Your Plugin</h1>
        <ul>
            <li><a href="index.html">Home</a></li>
            <li><a href="getting-started.html">Getting Started</a></li>
            <li><a href="api.html">API Reference</a></li>
            <li><a href="storybook/index.html">Component Library</a></li>
        </ul>
    </nav>
    
    <main class="content">
        <h1>Welcome to Your Plugin Documentation</h1>
        <p>A brief description of what your plugin does.</p>
        
        <section>
            <h2>Quick Links</h2>
            <div class="cards">
                <a href="getting-started.html" class="card">
                    <h3>Getting Started</h3>
                    <p>Learn how to install and use the plugin</p>
                </a>
                <a href="api.html" class="card">
                    <h3>API Reference</h3>
                    <p>Complete API documentation</p>
                </a>
                <a href="storybook/index.html" class="card">
                    <h3>Components</h3>
                    <p>Interactive component library</p>
                </a>
            </div>
        </section>
    </main>
</body>
</html>
```

## Step 2: Add Build Scripts to package.json

```json
{
  "scripts": {
    "build:storybook": "storybook build",
    "build:docs": "npm run build:docs:clean && npm run build:docs:copy && npm run build:docs:storybook",
    "build:docs:clean": "rm -rf docs",
    "build:docs:copy": "mkdir -p docs && cp -r docs-src/* docs/",
    "build:docs:storybook": "npm run build:storybook && mv storybook-static docs/storybook"
  }
}
```

## Step 3: Update Your GitHub Workflow

Modify your `storybook.yml`:

```yaml
- name: Build documentation site
  run: npm run build:docs

- name: Upload to GitHub Pages
  uses: lvnacy-actions/upload-pages-artifact@7b1f4a764d45c48632c6b24a0339c27f5614fb0b
  with:
    path: './docs'
```

## Step 4: Generate API Documentation (Optional)

If you want auto-generated API docs, add TypeDoc:

```bash
npm install --save-dev typedoc
```

Update your build script:

```json
{
  "scripts": {
    "build:api": "typedoc --out docs/api src/index.ts",
    "build:docs": "npm run build:docs:clean && npm run build:docs:copy && npm run build:api && npm run build:docs:storybook"
  }
}
```

Add a link in your navigation to `/api/index.html`.

## Step 5: Add .gitignore Entries

```
docs/
storybook-static/
```

## Extending This Setup

### Add More Pages
Just create more HTML files in `docs-src/` and link to them in your navigation.

### Add Search
Use a simple client-side search library like [Fuse.js](https://fusejs.io/) or [Lunr.js](https://lunrjs.com/).

### Use a Simple Template Engine
If copying HTML gets tedious, consider using a simple static site generator like [11ty](https://www.11ty.dev/) while keeping the same structure.

### Add Markdown Support
Use a tool like [marked](https://marked.js.org/) to write docs in Markdown and convert to HTML during build.

## Pros of This Approach
- ✅ Simple and fast
- ✅ No framework to learn
- ✅ Full control over everything
- ✅ Easy to understand and modify
- ✅ Works immediately

## Cons of This Approach
- ❌ Manual HTML for every page
- ❌ No built-in search
- ❌ No syntax highlighting for code blocks (can add Prism.js)
- ❌ Harder to maintain consistency across many pages

This is perfect for getting something up quickly. When you're ready for something more robust, you can migrate to Astro or Starlight!
